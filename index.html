Exynap

Easier Android Development

Exynap is an Android Studio plugin
that helps you to find and implement the required code

Just type whatever you are looking for

You only have to formulate a command, pick one suggestion and press Enter.

Easy to use

You do not have to remember tons of shortcuts or Live Template abbrevations. There is only one shortcut to remember (Ctrl + R). Nothing else.

Ready to use within minutes

Just download the plugin from the plugin repository and start. No sign up required.

Privacy guaranteed

A „Data Privacy Monitor“ shows you every data that has been submitted since Android Studio start. Exynap will never send implementation details, code or something project related to the backend.

Installation

  1. Open Android Studio
  2. Open the Settings dialog and select „Plugins“ in the left-hand pane
  3. Click „Browse repositories“ and enter „Exynap“ in the search field.
  4. Select „Exynap“ and click on „Install plugin“
  5. Confirm your intention to install Exynap and restart Android Studio

After the restart of Android Studio, you will see a „Quick Start Guide“ dialog. Please follow this short tutorial. It will help you to understand Exynap, so you can be more productive. Please do also take a look at the following, more detailed introduction to Exynap.

First Steps

When using Exynap, the first thing you have to do is to place your cursor inside the Java method where you want to insert the code.
In most cases it makes sense to place the cursor in an empty line.

There are only two cases when Exynap does not require an empty line:
Case #1: When the selected variable should be used for instance method calls:.

Case #2: When the evaluated result of Exynap should be assigned to a variable:

In every other case has the cursor to be placed in an empty row! (A future release will support insertion inside statements)

If a smart code insertion needs further input, Exynap will automatically place the cursor to the position that requires your input. Just type the value you want and press „Tab“. If you do not want to fill the placeholder you can press „Esc“. Exynap has adopted the behaviour of Intellij Live Templates.

It is absolutely necessary that you enter a command (e.g. „set repeating alarm“) and not a common search query (e.g. „how to set a repeating alarm“) in the input box.

Important: A command does always start with a verb!
Examples:

  • hide softkeyboard
  • create time picker dialog
  • set Actionbar title

If your command (e.g. „calculate the answer of life“) can not be interpreted by Exynap, you can press „Ctrl + Enter“ to trigger a Google search. Exynap helps you with the usage of the Android platform API. External libraries and the "core" Java API are not supported yet.
You can always undo a code insertion by using the regular undo shortcut (Ctrl + Z)

Examples

The following table shows a few examples to give you an overview about the possibilities of Exynap:

Command Result
create alert dialog Creates a new AlertDialog
instantiate TextView Instantiates a TextView object via code
bind view Binds the selected XML layout file (either native Android binding or via Butterknife)
hide keyboard hides the opened keyboard
get preferences Get an instance of SharedPreferences
put string to SharedPreferences To write a String into SharedPreferences
create new Activity To create a new subclass of Activity
Create new Adapter To create a simple Adapter stub
